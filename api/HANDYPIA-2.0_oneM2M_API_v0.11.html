oneM2M API for managing CSEBase

oneM2M 표준에서 root에 해당하는 자원을 CSE 또는 CSEBase라고 하며,본 설명에서는 CSEBase id를 'base'로 가정한다. CSEBase인 'base'자원은 플랫폼 구동시 초기에 생성되며, 하위에 다양한 자원이 포함 될 수 있으며, 여기에서는 CSEBase 자원에 대한 프리미티브 별 이용 방법을 안내한다. base'자원 아래 생성될 수 있는 자원 유형으로는 'AE', 'Container', 'Subscription','Group', 'Node', 'AccessControlPolicy', 'RemoteCSE' 등으로 다양하다.

/base post get put delete

post

CSEBase자원의 하위 자원에 대한 생성 요청.

get

CSEBase 정보를 검색. Queryparameter 는 맨 앞에 '?'를 둠.

put

CSEBase 속성에 대한 업데이트(추가, 변경, 삭제).

delete

CSEBase 정보를 삭제.

post /base

CSEBase자원의 하위 자원에 대한 생성 요청.

  • Request
  • Response

Headers

  • X-M2M-Origin: required (string)

    Originator, From 으로 CSE-ID 또는 AE-ID 가 올 수 있음. 또한, 해당 ID는 accessControlPolicy 상에 등록 되어 있어야 하며, 현재는 Sxx, Cxx 등의 ID는 accessControlPolicy 상에 등록 되어 있음.

    Example:

    SaeX
  • X-M2M-RI: required (string)

    Request Identifier 로 요청을 식별하기 위해 부여.

    Example:

    RQI-0001
  • Content-Type: required (string)

    두 가지 정보를 함께 포함하고 있으며, ';' 로 구분하고, 1) 자원의 content-format 으로 XML 또는 JSON 이 가능. 2) 자원의 타입으로 ty=x 형태로 구성되며, 'x' 에 해당하는 enum 값은 oneM2M의 TS-0004 문서의 6.3.3.2.11 m2m:memberType 상에 정의 되어 있음.

    Example:

    application/json;ty=2

Body

Type: application/json

Example:

{
   "m2m:ae" : {
      "lbl" : "HANDYPIA WP",
      "apn" : "HANDYPIA_WP_Test",
      "api" : "Non_registered_App_ID",
      "poa" : "http://localhost:18080/wpstub",
      "rr" : false
   }
}

oneM2M API to discover resources

oneM2M 표준의 자원에 대한 discovery 이용 방법을 안내한다. discovery는 모든 oneM2M 자원에 대해 수행할 수 있으며, 대상 URI 를 포함한 그 하위의 자원에 대해 검색해 준다. base은 CSEBase 자원의 ID 이다.

/base?fu=1 get

get

CSEBase 정보를 검색. Queryparameter 는 맨 앞에 '?'를 둠.

get /base?fu=1

CSEBase 정보를 검색. Queryparameter 는 맨 앞에 '?'를 둠.

  • Request
  • Response

Headers

  • X-M2M-Origin: required (string)

    Originator, From 으로 CSE-ID 또는 AE-ID 가 올 수 있음. 또한, 해당 ID는 accessControlPolicy 상에 등록 되어 있어야 하며, 현재는 Sxx, Cxx 등의 ID는 accessControlPolicy 상에 등록 되어 있음.

    Example:

    SaeX
  • X-M2M-RI: required (string)

    Request Identifier 로 요청을 식별하기 위해 부여.

    Example:

    RQI-0001
  • content-type: (string)

    수신하고자 하는 content-format.

    Example:

    application/json

Query Parameters

  • fu: (string)

    filterUsage 의 사용 여부로, fu 가 1로 세팅되어 전달되면, 조건에 해당하는 자원들의 URI정보를 discovery 하여 제공함. fu 에 이어 다양한 discovery 관련 query parameter가 추가될 수 있음. 추가되는 query parameter들은 구분자로 '&'를 주며, 서로 다른 파라미터 간에는 AND 연산을, 동일한 파라미터 간에는 OR 연산을 수행. fu 파라미터는, 모든 자원에 대해 줄 수 있으며, 대상 URI 및 하위 자원에 대한 정보를 제공함.

    Example:

    1
  • rty: (string)

    fu=1 일 때 이용 가능하며, resource type 으로, 특정 자원의 타입으로 한정하여 검색. 자원 별 rty 값은 oneM2M의 TS-0004 문서의 6.3.3.2.11 m2m:memberType 상에 정의 되어 있음.

    Example:

    2
  • lbl: (string)

    fu=1 일 때 이용 가능하며, 자원의 속성 중 labels에 해당 하는 정보가 일치하는 자원에 대해 검색 하는 것으로, 각 자원은HANDYPIA WP oneM2M과 같은 형태로 자신의 정보를 기록하고, 'lbl=HANDYPIA&lbl=WP' 등과 같은 형태로 파라미터에 줄 수 있음. HANDYPIA 2.0 플랫폼에서는, lbl 속성을 하나 이상 사용하면, 시스템 설정에 따라 OR 또는 AND 연산을 수행함.

    Example:

    HANDYPIA
  • crb: (string)

    fu=1 일 때 이용 가능하며, Created Before 로, 특정 시간 이전에 생성된 자원에 대해 검색.

    Example:

    20151009
  • cra: (string)

    fu=1 일 때 이용 가능하며, Created After 로, 특정 시간 이후에 생성된 자원에 대해 검색.

    Example:

    20150930
  • ms: (string)

    fu=1 일 때 이용 가능하며, Modified Since 로, 특정 시간 이후로 변경이 발생한 자원에 검색.

    Example:

    20150901
  • us: (string)

    fu=1 일 때 이용 가능하며, Unmodified Since 로, 특정 시간 이후로 변경이 발생하지 않은 자원에 대해 검색.

    Example:

    20151012
  • exa: (string)

    fu=1 일 때 이용 가능하며, Expire After 로, 특정 시간 이후로 expire 되는 자원에 대해 검색, expirationTime 이라는 속성에 포함된 값으로, 해당 시간이 도래하면, 대상 자원은 플랫폼에서 삭제 됨.

    Example:

    20151231
  • exb: (string)

    fu=1 일 때 이용 가능하며, Expire Before 로, 특정 시간 이전에 expire 되는 자원에 대해 검색.

    Example:

    20151012
  • lim: (string)

    fu=1 일 때 이용 가능하며, 언급한 Query Parameter 를 통해 검색되는 결과에 대해 전달되는 결과를 제한 함. 즉, lim=5로 주게 되면, 검색된 URI가 6개여도 5개 까지만 결과로 알려줌.

    Example:

    5
